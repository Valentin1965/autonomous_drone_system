## На полі

### Запуск

1. **Pixhawk** увімкнено, GPS fix (3D).
2. **RPi** (якщо CV на борту): `bash scripts/run_variant2_rpi.sh`
3. **GCS**: `SYSTEM_CONFIG=config/system_gcs.yaml` + IP радіо в `connection_px4`.
4. Переконайтесь: профіль **px4**, **без** симулятора (`--full` не використовувати).

### Перший рух

1. Ручний режим → короткий рух / ARM тест.
2. 2–3 точки маршруту → **Старт** на низькій швидкості (0.5–1 м/с).
3. На останній точці — **Стоп**; за потреби **↩ Повернення**.

### Якщо OFFLINE на GCS

- Перевірте радіо / IP у `config/system_gcs.yaml`.
- RPi USB до FC зазвичай лишається — CV може працювати; GCS лише моніторинг.
- **Аварійна зупинка** + фізичний E-stop.

## Після сесії

- GCS → **Завантажити лог сесії** (`/api/diagnostics/session-log`).
- Збережіть експорт маршруту JSON.
- Логи RPi: `logs/` (якщо увімкнено в config).

---

## Типові помилки

| Симптом | Рішення |
|---------|---------|
| Жовта смуга «px4 + симулятор» | Не запускайте `--full` у полі; лише `run_variant2_gcs.sh` |
| Heartbeat > 5 с | Перевірте UDP порт 14550, firewall, кабель TELEM |
| Два MAVLink клієнти до FC | mavlink-router на RPi або лише USB (RPi) + radio (GCS) |
| CV рухає після падіння | Оновлений трекер зупиняє рух при fatal; натисніть STOP |
